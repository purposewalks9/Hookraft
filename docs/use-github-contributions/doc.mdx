## How it works

`useGithubContributions` fetches GitHub's public contribution graph HTML through a proxy route, parses every `<td data-date>` cell, and assembles the data into a typed week-by-week structure — the same 53-column grid GitHub renders on profile pages.

Because GitHub does not expose an unauthenticated public API for contribution data, the hook always goes through a proxy. By default it hits `/api/github-contributions` on your own origin. You can override this with the `proxyUrl` option.

The proxy must:

1. Accept `?username=` (and optionally `?year=`) as query parameters.
2. Fetch `https://github.com/users/{username}/contributions` (with `?from=YYYY-01-01&to=YYYY-12-31` when a year is supplied).
3. Return the raw HTML from that GitHub endpoint.

The hook then hands that HTML string to the browser's `DOMParser`, selects every `td[data-date]`, and reads `data-date`, `data-level`, and the inner text count off each cell.

### Data pipeline at a glance

```
GitHub HTML  →  DOMParser  →  dayMap (date → ContributionDay)
             →  Sunday-aligned week buckets (ContributionWeek[])
             →  streak calculation  →  ContributionData returned to you
```

**Sunday alignment** — before the first real day is pushed into week buckets, the hook checks its day-of-week (`.getDay()`) and backfills that many empty placeholder days so every week always starts on Sunday and has exactly 7 entries.

**Streak logic** — `computeStreaks` walks the flat sorted day list backward from today, skipping future dates. It increments a running counter for every day with `count > 0` and resets on any gap. The first non-zero run encountered (going backward) becomes `currentStreak`; the highest run ever reached in the full walk is `longestStreak`.

## useGithubContributions

```tsx
import { useGithubContributions } from "@hookraft/use-github-contributions"
```

```tsx
const { data, loading, error, refetch } = useGithubContributions({
  username: "torvalds",
  year:     2024,
  theme:    "github",
  onLoad:   (data) => console.log(`${data.totalContributions} contributions`),
})
```

The hook manages its own fetch lifecycle. `loading` is `true` immediately on mount and after every `refetch()` call. The fetch re-runs automatically whenever `username`, `year`, or `proxyUrl` change (these three are the `useCallback` deps). `theme` and `customColors` are passed through to the `ContributionCalendar` component only — they have no effect on the raw data the hook returns.

### Full example

```tsx
"use client"

import { useGithubContributions } from "@hookraft/use-github-contributions"

function ContribHeatmap({ username }: { username: string }) {
  const { data, loading, error, refetch } = useGithubContributions({ username })

  if (loading) return <p>Loading…</p>
  if (error)   return <p>Error: {error.message}</p>

  return (
    <div>
      <p>
        {data!.totalContributions} contributions this year ·{" "}
        {data!.currentStreak}d current streak ·{" "}
        {data!.longestStreak}d longest streak
      </p>

      {/* Refresh without remounting */}
      <button onClick={refetch}>Refresh</button>

      {/* Raw grid — bring your own UI */}
      <div style={{ display: "flex", gap: 3 }}>
        {data!.weeks.map((week, wi) => (
          <div key={wi} style={{ display: "flex", flexDirection: "column", gap: 3 }}>
            {week.days.map((day) => (
              <div
                key={day.date}
                title={`${day.count} on ${day.date}`}
                style={{
                  width:        11,
                  height:       11,
                  borderRadius: 2,
                  background: ["#ebedf0","#9be9a8","#40c463","#30a14e","#216e39"][day.level],
                }}
              />
            ))}
          </div>
        ))}
      </div>
    </div>
  )
}
```

## ContributionCalendar

Drop-in component that wraps `useGithubContributions` internally and renders the full heatmap — month labels, day labels (Mon / Wed / Fri), the block grid, and optional stat cards. The grid scrolls horizontally on narrow viewports and never wraps.

```tsx
import { ContributionCalendar } from "@hookraft/use-github-contributions"
```

```tsx
<ContributionCalendar
  username="torvalds"
  theme="halloween"
  blockSize={13}
  showStats
  showTooltip
  onContributionClick={(day) => console.log(day)}
/>
```

### How the layout is built

The component calculates a `blockStep = blockSize + blockGap` value and uses it consistently for both the column widths and the absolute positioning of month labels. Month labels are placed with `left: weekIndex * blockStep` so they always align to the column they describe. Only odd-indexed day labels (Mon, Wed, Fri) are rendered to match GitHub's default.

Colors are resolved at render time via `resolveColors(theme, customColors)`. For built-in themes it reads from the `THEMES` constant; for `"custom"` it uses your `customColors` object. The resolved five-stop map is applied per block as `background: colorMap[day.level]`.

## Setting up the proxy

The hook cannot call `github.com` directly from the browser due to CORS. You need a thin server-side route that forwards the request.

### Next.js App Router example

```ts
// app/api/github-contributions/route.ts
import { NextRequest } from "next/server"

export async function GET(req: NextRequest) {
  const username = req.nextUrl.searchParams.get("username")
  const year     = req.nextUrl.searchParams.get("year")

  if (!username) {
    return new Response("username is required", { status: 400 })
  }

  const url = new URL(`https://github.com/users/${username}/contributions`)
  if (year) {
    url.searchParams.set("from", `${year}-01-01`)
    url.searchParams.set("to",   `${year}-12-31`)
  }

  const res = await fetch(url.toString(), {
    headers: { "X-Requested-With": "XMLHttpRequest" },
  })

  if (!res.ok) {
    return new Response(await res.text(), { status: res.status })
  }

  const html = await res.text()
  return new Response(html, {
    headers: { "Content-Type": "text/html" },
  })
}
```

### Next.js Pages Router example

```ts
// pages/api/github-contributions.ts
import type { NextApiRequest, NextApiResponse } from "next"

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  const { username, year } = req.query

  if (!username || typeof username !== "string") {
    return res.status(400).send("username is required")
  }

  const url = new URL(`https://github.com/users/${username}/contributions`)
  if (year && typeof year === "string") {
    url.searchParams.set("from", `${year}-01-01`)
    url.searchParams.set("to",   `${year}-12-31`)
  }

  const ghRes = await fetch(url.toString(), {
    headers: { "X-Requested-With": "XMLHttpRequest" },
  })

  res.status(ghRes.status).send(await ghRes.text())
}
```
